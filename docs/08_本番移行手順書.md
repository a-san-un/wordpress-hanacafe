## 1. 必須ランタイム

### 1.1 PHP

- [ ] PHP 8.0以上（str_starts_with 使用のため）
- [ ] fileinfo, mbstring など一般的なWordPress必須拡張が有効

### 1.2 WordPress

- [ ] WordPress 6.3以上（wp_enqueue_script の strategy オプション利用）
- [ ] 推奨は 6.9系（プロジェクト文書の前提）
- [ ] パーマリンク設定が有効（投稿・CPTの遷移確認）

### 1.3 Node.js（開発時）

- [ ] Node.js 18以上を利用
- [ ] npm install が完了
- [ ] npm run lint が成功

---

## 2. 必須プラグイン・データモデル

### 2.1 ACF

- [ ] Advanced Custom Fields が有効
- [ ] 以下のマスター系ページが存在
  - [ ] common-info
  - [ ] access-info
  - [ ] menu-info
  - [ ] about-seats
- [ ] 主要フィールドグループが紐付いている
  - [ ] Hero用 pic
  - [ ] 共通 fallback 用 site_default_image
  - [ ] Access用 shop\_\* / seat_check_url / map系
  - [ ] Menu用 top_menu\_\* と menu投稿用詳細フィールド

### 2.2 CPT / Taxonomy

- [ ] post_type menu が登録済み
- [ ] taxonomy menu_category が登録済み
- [ ] menu_category に slug food, drink, dessert が存在
  - スラッグ値は get_hanacafe_menu_categories()（functions.php）で一元管理。カテゴリー追加時は同関数のみ修正すればよい。
- [ ] menu投稿が最低1件以上存在（トップ・一覧の表示確認用）

## 3. テーマアセット依存

### 3.1 必須ファイル

- [ ] assets/css/app.css が存在
- [ ] assets/js/main.js が存在
- [ ] assets/images/coming-soon.jpg が存在
- [ ] assets/images/map.png が存在
- [ ] assets/images/hero.jpg が存在

### 3.2 外部ネットワーク

- [ ] fonts.googleapis.com への疎通
- [ ] fonts.gstatic.com への疎通
- [ ] CSPやFW設定でGoogle Fontsがブロックされていない

---

## 4. テーマ機能前提

### 4.1 ナビゲーション

- [ ] global-nav にメニュー割り当て済み
- [ ] drawer-nav にメニュー割り当て済み
- [ ] ハンバーガー開閉、Esc閉鎖、フォーカストラップが動作

### 4.2 画像フォールバック

- [ ] アイキャッチ未設定投稿でデフォルト画像表示
- [ ] alt未設定画像でタイトル補完が動作

### 4.3 テンプレート分岐

- [ ] home投稿一覧が archive-post.php に切り替わる
- [ ] menuアーカイブが全件表示（posts_per_page=-1）

---

## 5. セキュリティ・運用品質

### 5.1 出力エスケープ

- [ ] 主要テンプレートで esc_html / esc_attr / esc_url が適用
- [ ] Access交通情報の許可HTMLが想定通り（wp_kses_post）

### 5.2 デバッグ・ログ

- [ ] WP_DEBUG, WP_DEBUG_LOG の運用方針を環境別に定義
- [ ] PHP Warning/Notice がフロントに表示されない設定

## 7. 本番移行手順（All-in-One WP Migration + FTP 直接アップロード併用）

### 事前準備（ローカル）

| # | 作業 |
|---|---|
| 7-1 | WordPress 本体を公式サイトからダウンロード・展開 |
| 7-2 | FileZilla で `wordpress/` の中身（`wp-config.php` 除く）を `/hanacafe/wp/` にアップロード |
| 7-3 | ローカルの `index.php` を `/hanacafe/` に、`.htaccess` を `/hanacafe/` にアップロード |
| 7-4 | テーマ一式を `/hanacafe/wp/wp-content/themes/hanacafe-theme/` にアップロード |
| 7-5 | `uploads/` を `/hanacafe/wp/wp-content/uploads/` にアップロード |

### 本番サーバー：WordPress 新規インストール

| # | 作業 |
|---|---|
| 7-6 | ブラウザで `https://intp.site/2999/WordPress/hanacafe/` にアクセス |
| 7-7 | DB情報を入力（テーブル接頭辞は `hc_` に変更） |
| 7-8 | インストール完了後、仮ユーザーで管理画面にログイン |

### All-in-One WP Migration：エクスポート（ローカル）

| # | 作業 |
|---|---|
| 7-9 | ローカル管理画面 → All-in-One WP Migration → エクスポート |
| 7-10 | URL置換: `http://hanacafe.local` → `https://intp.site/2999/WordPress/hanacafe` |
| 7-11 | 除外設定: メディアライブラリ・テーマ・プラグインを除外（必須プラグインは除外しない） |
| 7-12 | `.wpress` ファイルをダウンロード保存 |

### All-in-One WP Migration：インポート（本番）

| # | 作業 |
|---|---|
| 7-13 | 本番管理画面 → All-in-One WP Migration → インポート → `.wpress` を選択 |
| 7-14 | 強制ログアウト後、ローカルのユーザー名・パスワードで再ログイン |

### インポート後の設定・確認

| # | 作業 |
|---|---|
| 7-15 | 設定 → パーマリンク → 「変更を保存」を2回押す |
| 7-16 | 外観 → テーマ → `hanacafe-theme` を有効化 |
| 7-17 | 設定 → 表示設定 で「検索エンジンにインデックスさせない」が **OFF** か確認 |
| 7-18 | `wp-config.php` のパーミッションを 400 に変更（FTP） |
| 7-19 | `wp/.htaccess` に `wp-config.php` へのアクセス拒否コードを追記 |
| 7-20 | 本番URLでトップページ・各ページが正常に表示されるか確認 |
| 7-21 | パンくずリンクのURLに `/wp/` が混入していないか確認（サブディレクトリ構成時に発生しやすい） |
| 7-22 | SNSシェア時にOGP画像・タイトルが正しく表示されるか |
| 7-23 | お問い合わせ・予約リンクが正しく遷移するか |
| 7-24 | Google Search Console にサイトマップを送信したか |
