## 1. 環境・WordPress設定

| # | チェック項目 | 確認方法 |
|---|---|---|
| 1-1 | WordPress のバージョンが最新か | 管理画面「ダッシュボード」→「更新」 |
| 1-2 | プラグイン（ACF / CPT UI 等）が最新か | 管理画面「プラグイン」→「更新」 |
| 1-3 | パーマリンク設定が「投稿名」になっているか | 設定 → パーマリンク → 「投稿名」を選択 → 保存 |
| 1-4 | 「検索エンジンがサイトをインデックスしないようにする」が **OFF** になっているか | 設定 → 表示設定 → チェックボックスを確認 |
| 1-5 | サイトURL・WordPressURLが本番ドメインになっているか | 設定 → 一般 |
| 1-6 | 管理者メールアドレスが正しいか | 設定 → 一般 |
| 1-7 | デフォルトのサンプルページ・投稿が削除されているか | 固定ページ・投稿一覧を確認 |

### PC（ブラウザ）

| # | チェック項目 | 確認ページ |
|---|---|---|
| 3-11 | トップページが崩れていないか | `/` |
| 3-12 | スクロール前: ヘッダー背景が `rgba($c-base, 0.95)`（生成色95%）か。スクロール後: `.is-scrolled` が付与され、背景が純白・下端に `rgba($c-text, 0.1)` の境界線が出現するか（DevTools → Elements で `.l-header.is-scrolled` を確認） | `/` |

## 4. セキュリティ確認

| # | チェック項目 | 確認方法 |
|---|---|---|
| 4-1 | WordPress のデバッグモードが **OFF** になっているか | `wp-config.php` の `WP_DEBUG` が `false` |
| 4-2 | 管理者パスワードが十分に強いか（辞書攻撃耐性） | WordPress ユーザー設定で確認 |
| 4-3 | 不要なユーザーアカウントが削除されているか | 管理画面「ユーザー」一覧 |
| 4-4 | ファイルパーミッションが適切か（`wp-config.php` は 400） | サーバー設定・FTPで確認 |
| 4-5 | XML-RPC が不要な場合は無効化されているか | プラグイン等で確認 |

## 5. SEO・アクセシビリティ確認

| # | チェック項目 | 確認方法 |
|---|---|---|
| 5-1 | 各ページのタイトルタグが適切に設定されているか | ブラウザタブ・ソース確認 |
| 5-2 | OGP（og:title / og:image）が設定されているか | ソース or SNSデバッガーで確認 |
| 5-3 | 全画像に `alt` 属性が設定されているか（自動補完機能の動作確認） | Chrome DevTools |
| 5-4 | メニュー投稿・席情報カードの `alt` が空でないか | 各ページを目視確認 |
| 5-5 | 見出し階層（h1→h2→h3）が論理的に構成されているか | Chrome DevTools |
| 5-6 | キーボード操作でナビゲーションが操作できるか | Tab キーで確認 |
| 5-7 | Yoast SEO がインストール・有効化されているか | 管理画面「プラグイン」 |
| 5-8 | XMLサイトマップが生成されているか（/sitemap_index.xml） | ブラウザで直接アクセス |
| 5-9 | OGP画像がYoast SEOで設定されているか | Yoast SEO → ソーシャル |
| 5-10 | 投稿者アーカイブが無効化されているか | Yoast SEO → 詳細設定 |

## 7. 本番移行手順（All-in-One WP Migration + FTP 直接アップロード併用）

### 事前準備（ローカル）

| # | 作業 |
|---|---|
| 7-1 | WordPress 本体を公式サイトからダウンロード・展開 |
| 7-2 | FileZilla で `wordpress/` の中身（`wp-config.php` 除く）を `/hanacafe/wp/` にアップロード |
| 7-3 | ローカルの `index.php` を `/hanacafe/` に、`.htaccess` を `/hanacafe/` にアップロード |
| 7-4 | テーマ一式を `/hanacafe/wp/wp-content/themes/hanacafe-theme/` にアップロード |
| 7-5 | `uploads/` を `/hanacafe/wp/wp-content/uploads/` にアップロード |

### 本番サーバー：WordPress 新規インストール

| # | 作業 |
|---|---|
| 7-6 | ブラウザで `https://intp.site/2999/WordPress/hanacafe/` にアクセス |
| 7-7 | DB情報を入力（テーブル接頭辞は `hc_` に変更） |
| 7-8 | インストール完了後、仮ユーザーで管理画面にログイン |

### All-in-One WP Migration：エクスポート（ローカル）

| # | 作業 |
|---|---|
| 7-9 | ローカル管理画面 → All-in-One WP Migration → エクスポート |
| 7-10 | URL置換: `http://hanacafe.local` → `https://intp.site/2999/WordPress/hanacafe` |
| 7-11 | 除外設定: メディアライブラリ・テーマ・プラグインを除外（必須プラグインは除外しない） |
| 7-12 | `.wpress` ファイルをダウンロード保存 |

### All-in-One WP Migration：インポート（本番）

| # | 作業 |
|---|---|
| 7-13 | 本番管理画面 → All-in-One WP Migration → インポート → `.wpress` を選択 |
| 7-14 | 強制ログアウト後、ローカルのユーザー名・パスワードで再ログイン |

### インポート後の設定・確認

| # | 作業 |
|---|---|
| 7-15 | 設定 → パーマリンク → 「変更を保存」を2回押す |
| 7-16 | 外観 → テーマ → `hanacafe-theme` を有効化 |
| 7-17 | 設定 → 表示設定 で「検索エンジンにインデックスさせない」が **OFF** か確認（1-4） |
| 7-18 | `wp-config.php` のパーミッションを 400 に変更（FTP） |
| 7-19 | `wp/.htaccess` に `wp-config.php` へのアクセス拒否コードを追記 |
| 7-20 | 本番URLでトップページ・各ページが正常に表示されるか確認（セクション3） |
| 7-21 | パンくずリンクのURLに `/wp/` が混入していないか確認（サブディレクトリ構成時に発生しやすい） |
| 7-22 | SNSシェア時にOGP画像・タイトルが正しく表示されるか |
| 7-23 | お問い合わせ・予約リンクが正しく遷移するか |
| 7-24 | Google Search Console にサイトマップを送信したか |
