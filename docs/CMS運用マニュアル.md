## 1. メニューを追加・更新する

### 概要

メニュー（フード・ドリンク・デザート）は「メニュー」カスタム投稿タイプで管理します。
各メニュー投稿にACFフィールドで価格・説明・画像などを設定します。

### 手順

#### 1-1. 新しいメニューを追加する

1. WordPress 管理画面にログイン
2. 左メニュー「メニュー」→「新規追加」をクリック
3. 以下の項目を設定する

| 項目 | 説明 | 必須 |
|------|------|------|
| タイトル | メニュー名（例: シーズナルパフェ） | ✅ |
| アイキャッチ画像 | メインのメニュー写真 | 推奨 |
| カテゴリー（menu_category） | food / drink / dessert から選択 | ✅ |

4. 画面右下の「ACFフィールド」セクションを入力する

| フィールド名 | 入力内容 | 未入力時の挙動 |
|---|---|---|
| sub_name | サブタイトル（例: seasonal parfait） | 非表示 |
| price | 価格（例: 980） | 非表示 |
| is_recommended | チェックを入れるとRECOMMENDバッジが表示される | バッジ非表示 |
| menu_sub_img | サブ画像（詳細ページ用） | アイキャッチ→共通デフォルト画像の順でフォールバック |

5. 右上の「公開」ボタンをクリックして保存する

#### 1-2. 既存メニューを更新する

1. 左メニュー「メニュー」→「メニュー一覧」をクリック
2. 更新したい投稿名をクリックして編集画面を開く
3. 変更後、「更新」ボタンをクリックして保存する

#### 1-3. トップページへの表示設定（ピックアップ）

トップページに表示される代表メニュー（Food・Drink・Dessert 各 1 件）は、
固定ページ「【マスター】メニュー設定」（スラッグ: `menu-info`）のACFフィールドで手動指定します。

1. 左メニュー「固定ページ」→「固定ページ一覧」をクリック
2. スラッグが `menu-info` の固定ページ（【マスター】メニュー設定）を開く
3. ACFフィールド / top_menu_meal / top_menu_drink / top_menu_dessert」で表示したいメニュー投稿をそれぞれ選択する
4. 「更新」ボタンをクリックして保存する

> ⚠️ `menu-info` を未設定のままにすると、各カテゴリーの最新投稿が自動表示されます（フォールバック動作）。

## 2. 席情報を更新する

### 概要

席情報（カウンター・テーブル・テラス）は、固定ページ「【マスター】席情報・About」（スラッグ: `about-seats`）の
ACFフィールドで管理します。満席・空席のステータスもここから変更できます。

### 手順

#### 2-1. 席の空き状況を更新する

1. 左メニュー「固定ページ」→「固定ページ一覧」をクリック
2. スラッグが `about-seats` の固定ページ（【マスター】席情報・About）を開く
3. 変更したい席種のACFフィールドを探す

| 席種 | ステータスフィールド名 |
|------|----------------------|
| カウンター | status_counter |
| テーブル | status_table |
| テラス | status_terrace |

4. ステータスを変更する（例: `ok` → `full` など）
5. 「更新」ボタンをクリックして保存する

#### 2-2. 席の説明文・画像を更新する

各席種には以下のフィールドがあります。

| フィールド名 | 内容 | 未入力時の挙動 |
|---|---|---|
| title_[席種] | 席名（例: Counter） | カードが非表示になる |
| text_[席種] | 席の説明文 | 非表示 |
| img_[席種] | 席の写真 | `assets/images/{seats}.jpg` にフォールバック |
| is_pet_terrace | Pet Friendly バッジ表示のON/OFF（**テラス席のみ**） | バッジ非表示 |

> ⚠️ `title_[席種]` が空の場合、そのカード自体が非表示になります。必ず入力してください。
> ⚠️ `is_pet` フォールバックフォールドは **テラス席（`is_pet_terrace`）のみ** ACF に定義されています。カウンター席・テーブル席の管理画面には表示されません。

#### 2-3. 「Aboutセクション」の見出し・本文を変更する

1. `about-seats` 固定ページ（【マスター】席情報・About）を開く
2. 以下のフィールドを編集する

| フィールド名 | 内容 |
|---|---|
| about_section_title | Aboutセクションの見出し |
| about_section_text | Aboutセクションの本文 |

## 3. ニュースを投稿する

### 概要

お知らせ・ニュースは「投稿」（通常のWordPress投稿）で管理します。
カテゴリーや日付が自動でトップページ・ニュース一覧に反映されます。

### 手順

#### 3-1. 新しいニュースを投稿する

1. 左メニュー「投稿」→「新規追加」をクリック
2. 以下の項目を設定する

| 項目 | 説明 | 必須 |
|------|------|------|
| タイトル | ニュースの見出し（例: 夏季限定メニュー開始のお知らせ） | ✅ |
| 本文 | ニュースの詳細内容 | 推奨 |
| カテゴリー | 該当するカテゴリーを選択（なければ「一般」） | 推奨 |
| アイキャッチ画像 | サムネイル画像（一覧ページで表示される） | 推奨 |

3. 右上の「公開」ボタンをクリックして保存する

> 📌 投稿日時は自動で入ります。過去日付に設定したい場合は、「公開」横の日時をクリックして変更してください。

#### 3-2. 投稿を非公開・下書きに戺す

1. 対象の投稿を開く
2. 右サイドバー「公開状態」をクリック → 「下書き」または「非公開」を選択
3. 「更新」ボタンをクリックして保存する

## 運用上の注意事項

- **店舗情報（住所・電話番号・営業時間・SNS）の変更** は固定ページ `access-info`（【マスター】店舗情報）のACFフィールドから行う。直接PHPファイルを編集してはいけない（SSOT設計）。
- **ヒーロー画像の変更** は固定ページ `common-info`（【マスター】共通設定）のACFフィールド `pic` から行う。
- **トップメニューのピックアップ変更** は固定ページ `menu-info`（【マスター】メニュー設定）から行う。
- メニューカテゴリーのスラッグ（`food` / `drink` / `dessert`）は変更しないこと。変更するとトップページのフォールバック表示が壊れる。
- **パンくずリスト** はプラグインにより自動表示される。表示対象は個別ページ（メニュー詳細・ニュース詳細）およびアーカイブページ。トップページには表示されない。

## 5. 画像サイズ・命名規則

### 5-1. 推奨画像サイズ

| 用途 | 推奨サイズ | 備考 |
|---|---|---|
| ヒーロー画像（`common-info` > `pic`） | 1920×1080px 以上 | 横長・高解像度推奨 |
| メニューアイキャッチ | 800×800px | 正方形を推奨（カード表示でトリミングされる） |
| 席種画像（`img_[席種]`） | 800×600px | 横長推奨 |
| 地図画像（`shop_map_image`） | 800×600px | PNG または JPEG |
| ニュースアイキャッチ | 1200×630px | OGP用サイズを推奨 |

### 5-2. 画像ファイル命名規則

- 使用可能な文字: 半角英数字・ハイフン（`-`）のみ
- スペース禁止: スペースを含むファイル名はURLエンコードされてリンク切れの原因になる
- 大文字禁止: すべて小文字で統一する

| NG | OK |
|---|---|
| `Menu Photo.jpg` | `menu-photo.jpg` |
| `SeasonalPaffe2026.png` | `seasonal-parfait-2026.png` |
| `席写真_counter.jpg` | `seat-counter.jpg` |

### 5-3. 画像アップロード手順

1. WordPress 管理画面「メディア」→「新規追加」
2. ファイルを選択してアップロード
3. アップロード後、各フィールドの「画像を選択」から紐付ける

> 📌 一度アップロードしたファイルは「メディアライブラリ」から再利用できます。
> 同じ画像を重複アップロードしないようにしてください。
