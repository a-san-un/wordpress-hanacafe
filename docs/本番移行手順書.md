# HanaCAFE 本番移行手順書

HanaCAFE WordPress サイトをローカル環境から本番環境へ移行するための手順書です。
WordPress本体・テーマ・プラグイン・ACF/SEO設定を安全に引き継ぐことを目的とします。

> 公開前のコンテンツ・デザイン確認（ACFデータ・表示確認・SEO等）は
> [品質チェックリスト.md](品質チェックリスト.md) を参照してください。

---

## 1. 前提条件

### 1.1 サーバー要件

- PHP 8.0 以上（PHP 8.0 関数 `str_starts_with` を使用）
- PHP拡張: `fileinfo`, `mbstring`

### 1.2 WordPress バージョン

- WordPress 6.3 以上（`wp_enqueue_script()` の `strategy` 引数対応）
- 推奨: 6.9 系

### 1.3 Node.js / npm

- Node.js 18 以上
- `npm install` 実行済み
- `npm run lint` でエラーなし

---

## 2. 事前確認

### 2.1 ACF 設定

以下の固定ページが存在し、ACF設定が投入されていることを確認する。

- `common-info`
- `access-info`
- `menu-info`
- `about-seats`

確認対象:
- Hero: `pic`
- fallback: `site_default_image`
- Access: `shop_*`, `seat_check_url`, `shop_map_*`
- Menu: `top_menu_*`

### 2.2 CPT / Taxonomy

- post type: `menu`
- taxonomy: `menu_category`
- `menu_category` の slug は `meal`, `drink`, `dessert`
- `get_hanacafe_menu_categories()`（`functions.php`）と一致していること
- 各カテゴリーに最低1件以上のメニューがあること

---

## 3. ファイル確認

### 3.1 必須アセット

- `assets/css/app.min.css`
- `assets/js/main.js`
- `assets/images/coming-soon.jpg`
- `assets/images/map.png`
- `assets/images/hero.jpg`

### 3.2 外部依存

- `fonts.googleapis.com`
- `fonts.gstatic.com`
- CSP / Firewall で Google Fonts が遮断されないこと

---

## 4. 動作確認

### 4.1 ナビゲーション

- `global-nav`
- `drawer-nav`
- `Esc` でドロワーが閉じること

### 4.2 画像

- 画像切れがないこと
- alt が空でないこと

### 4.3 テンプレート

- home は `archive-post.php` を使用すること
- menu は `posts_per_page = -1` で全件表示されること

### 4.4 メニュー表示

- Menu セクションおよびアーカイブで `meal` / `drink` / `dessert` の3カテゴリが表示されること
- トップページの `top_menu_meal` / `top_menu_drink` / `top_menu_dessert` が正しく表示されること

---

## 5. セキュリティ・品質

### 5.1 エスケープ

- `esc_html`, `esc_attr`, `esc_url` の適用確認
- Access の一部HTMLは `wp_kses_post` を使用していること

### 5.2 デバッグ設定

- `WP_DEBUG`, `WP_DEBUG_LOG` は本番でOFF
- PHP Warning / Notice が表示されないこと

---

## 6. 表示確認ポイント

- Hero
- About（3席種）
- Menu（3カテゴリ）
- Access
- News
- Footer
- `meal` / `drink` / `dessert` の順序が保持されていること
- 地図オーバーレイのボタン文言が `Google Maps` になっていること
- SNS / OGP / 構造化データに欠落がないこと

## 7. 本番移行手順（All-in-One WP Migration + FTP 直接アップロード併用）

### 事前準備（ローカル）
| # | 作業 |
|---|---|
| 7-1 | WordPress 本体を公式サイトからダウンロード・展開 |
| 7-2 | FileZilla で `wordpress/` の中身（`wp-config.php` 除く）を `/hanacafe/wp/` にアップロード |
| 7-3 | ローカルの `index.php` を `/hanacafe/` に、`.htaccess` を `/hanacafe/` にアップロード |
| 7-4 | テーマ一式を `/hanacafe/wp/wp-content/themes/hanacafe-theme/` にアップロード |
| 7-5 | `uploads/` を `/hanacafe/wp/wp-content/uploads/` にアップロード |

### 本番サーバー：WordPress 新規インストール
| # | 作業 |
|---|---|
| 7-6 | ブラウザで `https://intp.site/2999/WordPress/hanacafe/` にアクセス |
| 7-7 | DB情報を入力（テーブル接頭辞は `hc_` に変更） |
| 7-8 | インストール完了後、仮ユーザーで管理画面にログイン |

### All-in-One WP Migration：エクスポート（ローカル）
| # | 作業 |
|---|---|
| 7-9 | ローカル管理画面 → All-in-One WP Migration → エクスポート |
| 7-10 | URL置換: `http://hanacafe.local` → `https://intp.site/2999/WordPress/hanacafe` |
| 7-11 | 除外設定: メディアライブラリ・テーマ・プラグインを除外（必須プラグインは除外しない） |
| 7-12 | `.wpress` ファイルをダウンロード保存 |

### All-in-One WP Migration：インポート（本番）
| # | 作業 |
|---|---|
| 7-13 | 本番管理画面 → All-in-One WP Migration → インポート → `.wpress` を選択 |
| 7-14 | 強制ログアウト後、ローカルのユーザー名・パスワードで再ログイン |

### インポート後の設定・確認
| # | 作業 |
|---|---|
| **7-15** | **【超重要】設定 → パーマリンク → 内容は変更せずに「変更を保存」を押す（リライトルールの再生成。忘れると /news/ 個別記事が404エラーになります）** |
| 7-16 | 外観 → テーマ → `hanacafe-theme` を有効化 |
| 7-17 | 設定 → 表示設定 で「検索エンジンにインデックスさせない」が **OFF** か確認 |
| 7-18 | `wp-config.php` のパーミッションを 400 に変更（FTP） |
| 7-19 | `wp/.htaccess` に `wp-config.php` へのアクセス拒否コードを追記 |
| 7-20 | 本番URLでトップページ・各ページが正常に表示されるか確認 |
| 7-21 | パンくずリンクのURLに `/wp/` が混入していないか確認（サブディレクトリ構成時に発生しやすい） |
| 7-22 | SNSシェア時にOGP画像・タイトルが正しく表示されるか |
| 7-23 | お問い合わせ・予約リンクが正しく遷移するか |
| 7-24 | Google Search Console にサイトマップを送信したか |
