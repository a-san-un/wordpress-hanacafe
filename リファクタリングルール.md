## 📋 HanaCAFE nappa69 リファクタリング＆実装ガイドライン

**【AIメンターへの指示】**
以下のルールと設計思想は、HanaCAFEプロジェクトにおける「絶対の憲法（SSOT）」です。今後のコード生成・修正は、必ずこのガイドラインに準拠して行ってください。

### 1. リファクタリングの「5つの黄金律」（絶対遵守ルール）

静的コードをWordPressテーマ化する際は、以下のルールでコードを浄化すること。

1. **インラインスタイルと不要タグの完全排除【HTML】**
* `style="..."` 属性は全削除。
* 無駄な `<div>` を減らし、WP標準ブロック互換のセマンティックなタグ（`section`, `figure` 等）へ置換する。


2. **SSOTに基づく変数化【SCSS】**
* 生のカラーコード（例: `#2E4D07`）を直書きしない。必ず `_variables.scss` の変数へ強制マッピングする。


3. **流体UIと共通レイアウトの死守【レスポンシブ】**
* `position: absolute;` や固定幅による強引な配置を排除し、Flexbox / CSS Gridを使用する。
* 左右の安全余白は共通コンテナ（PC 80px / SP 24px）を継承する。


4. **DRY原則に基づく動的対応【PHP / WordPress】**
* 同じHTML構造を `if / else` で繰り返さない。変数を先に定義し、HTMLの出力は1箇所にまとめる。
* コンテンツ部分は `the_title()` や ACFの `get_field()` で置換・ループ化しやすいタグ構造にする。


5. **アクセシビリティ（A11Y）の担保**
* ボタンやリンクの `:focus-visible` を設定する。



### 2. 共通レイアウト設計：コンテナと余白の考え方

「Food Science」の設計ルールを踏襲し、サイト全体の横幅と余白を管理する。

* **3つの幅の使い分け**:
* **通常幅 (800px)**: 読ませるテキスト（ブログ記事、ニュース詳細）。
* **広幅 (1200px)**: 共通クラス `.l-container` を使用（メインコンテンツ、ギャラリー）。
* **全幅 (100vw)**: 共通クラス `.u-alignfull` を使用。


* **全幅表示（ブレイクアウト）の実装**:
親コンテナを突き破る際は以下のCSSロジックを使用する。
`width: 100vw; margin-left: calc(50% - 50vw); margin-right: calc(50% - 50vw); max-width: none;`
* **境界と余白（脱・線まみれ）**:
セクションの境界は「線」ではなく「十分な余白（PC 120px / SP 60px目安）」と「背景色の交互切り替え（白 ⇔ ごく薄いグレー等）」で表現する。

### 3. プロフェッショナル実装テクニック（Food Scienceからの応用）

標準的な実装から一歩踏み込んだ、HanaCAFE向けの最適化手法。

* **JSの負荷軽減（スクロール検知）**:
スクロールごとに発火する重い処理を防ぐため、ヘッダーの背景変化（透過 → 白っぽく変化）などのスクロールイベントには必ず `requestAnimationFrame` を用いて負荷軽減（throttle処理）を行う。また、1pxではなく `> 20px` などで発火させる。
* **アクセシビリティのJS連動**:
ハンバーガーメニュー実装時は、jQueryで必ず `aria-expanded` の `true/false` 切り替え制御コードを含める。
* **Sass変数とtheme.jsonのハイブリッド運用**:
SCSSの関数（`rgba()`等）でのエラーを防ぐため、色のマスターは `_variables.scss` のHEXコードとするが、WPブロックエディタ側でも同じ色が使えるよう `theme.json` にパレットを同期させる。
* **No Image処理の確実な実装**:
投稿一覧等で `has_post_thumbnail()` が無い場合は、必ずデフォルト画像のパスを出力するフォールバックを実装する（共通関数化を推奨）。
* **写真の魅力を最大化するCSSテクニック**:
* **乗算フィルター**: ただ黒い膜を被せるのではなく `mix-blend-mode: multiply;` を使い、背景画像にブランドカラー（深緑等）を溶け込ませる。
* **構図の固定**: スマホとPCの縦横比の違いを考慮し、`object-fit: cover;` の画像にはデバイスごとに最適な `object-position` をメディアクエリで設定する。


* **テキストの中央配置 ＋ 左揃え**:
絶対配置を避けつつ「画面中央に配置するが、テキスト自体は左揃え」を実現する場合は、ラッパーに `width: fit-content; margin: 0 auto;` を使用する。
