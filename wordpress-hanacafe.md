# 📋 HanaCAFE nappa69 サイトリニューアル制作要件＆環境定義（SSOT）

Web クリエイター・エンジニア科の課題「HanaCAFE nappa69 サイトリニューアル」における、技術スタックおよびベースとなるコーディング規約・バージョン管理の絶対基準ファイルです。

---

## 1. プロジェクト概要と技術スタック
* **CMS**: WordPress (バージョン 6.6+)
* **CSSメタ言語**: Dart Sass (最新仕様 `@use` 準拠)
* **CSS設計**: FLOCSS + BEMベース (モバイルファースト設計)
* **JavaScript**: Vanilla JS (モダンES6+) および WordPress内包の jQuery
* **主要プラグイン**: Advanced Custom Fields (ACF) PRO相当のフィールド設計

※ **具体的な実装ルール（黄金律）やファイルディレクトリ構造については、`architecture-guidelines.md` を参照すること。**

## 2. バージョン管理（Git コミット規約）

コミット履歴の可読性を高めるため、コミットメッセージは「Conventional Commits」に準拠し、以下のプレフィックス（接頭辞）から開始すること。

* `feat:` 新しい機能の追加（例: ACFのフィールド追加、新セクションの実装）
* `fix:` バグ修正（例: 表示崩れの解消、リンク切れの修正）
* `refactor:` リファクタリング（機能追加やバグ修正を含まない、コードの構造改善・整理）
* `style:` スタイルのみの調整（CSSの微調整、フォーマット修正、インデント修正）
* `docs:` ドキュメントのみの更新（例: READMEやマークダウンファイルの修正）
* `chore:` ビルドタスク、パッケージマネージャーの設定、画像圧縮などの雑務

**コミットメッセージの例**:
> refactor: バッジスタイルのコンポーネント化と重複コードの排除
> feat: AboutセクションのACFお席状況フィールドを実装

## 3. ベース・コーディング規約

### PHP / WordPress
* **インデント**: タブ文字（スペースではない）。
* **括弧のスタイル**: K&Rスタイル（関数・制御構文と同じ行に `{` を配置）。
* **命名規則**:
  * 関数名・変数名: スネークケース（`$my_variable`, `my_function`）
  * クラス名: パスカルケース（`class My_Class_Name`）
  * 定数: 大文字スネークケース（`MY_CONSTANT`）
* **セキュリティ（必須）**: データ出力時は必ず `esc_html()`, `esc_url()`, `esc_attr()` などの遅延エスケープを行う。
* **ACF取得**: データ取得時は `get_field('フィールド名')` を使用する。

### SCSS / CSS
* **モダンSass**: Dart Sass の仕様に基づき、`@use` ルールは必ずファイルの最上部に記述する。
* **状態クラス**: JS等で状態を操作するクラス名は `is-` プレフィックス（例: `.is-active`, `.is-scrolled`）を用いる。
* **カラーと数値**: 色コード（`#FFF`等）や共通の数値は直接記述せず、必ず `_variables.scss` で定義した変数（`v.$c-xxx`）を使用する。

### HTML / JavaScript
* **変数宣言**: 原則として `const` を使用し、再代入が必要な場合のみ `let` を使用する（`var` は使用禁止）。
* **命名規則**:
  * 変数名・関数名: キャメルケース（`myVariable`, `myFunction`）
* **JSフック**: イベントフック用の要素には、スタイリング用のクラスとは別に必ず `.js-hamburger` などの `js-` プレフィックスを付与する。
* **jQuery使用時**: WordPress内でjQueryを使用する場合は、コンフリクトを避けるため必ず `jQuery(function($){...})` でラップする。
